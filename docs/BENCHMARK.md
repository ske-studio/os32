# OS32 Benchmark (BENCH.BIN)

OS32システムにおける統合ベンチマークプログラム仕様書

## 概要

`bench.bin` は、OS32の各サブシステム（メモリマネージャ、グラフィックス、テキストレンダリング、ファイルシステム等）が正常に動作し、期待されるパフォーマンスを発揮できるかを統合的に検証・評価するためのテストプログラムです。

実行完了後、結果はコンソールに出力されると同時に、仮想ファイルシステムを通じて `0:/bench.log` というテキストファイルに保存されます。

## テスト項目一覧

### 1. Memory & CPU Test (`test_memory`)
メモリ（ヒープ）のアロケーション機能と、簡単なCPUテスト（memsetの速度）を評価します。
* 処理内容: 4KBごとの動的メモリ確保（`mem_alloc`）、値クリア（`memset`）、および解放（`mem_free`）を1,000回繰り返します。

### 2. Graphics (GFX) Test (`test_gfx`)
GDC（グラフィックディスプレイコントローラ）およびバックバッファ層での描画性能を直接評価します。
* 処理内容: GFXサブシステムの初期化後、バックバッファへの描画（直線・矩形塗りつぶしをそれぞれ500個生成）と物理VRAMへの転送（`gfx_present`）を行います。描画プリミティブは `libos32gfx` ライブラリまたはフレームバッファ直接操作で実行されます。テスト終了後にGFXリソースを破棄します。

### 3. KCG Text Rendering Test (`test_kcg`)
KCG（漢字ROM）からのフォント読み出し、およびUTF-8からShift_JISへの変換オーバーヘッドとテキストレンダリング速度を評価します。
* 処理内容: 画面全域に対して `kcg_draw_utf8` を用い、ASCIIおよびマルチバイト文字を含むテキストを連続してバックバッファに描画します。

### 4. VFS (File System) Test (`test_vfs`)
VFSのスタックおよびFAT12ドライバなどのストレージIOの負荷・レイテンシを評価します。
* 処理内容: ファイル（`0:/bench.tmp`）を新規作成（`KAPI_O_CREAT`）し、ダミーデータ書き込み（50回）と読み出し（50回）を行い、最後に `sys_unlink` で削除します。

### 5. FM Sound Test (`test_fm`)
FM（OPN）音源のMMLパースおよび再生初期化への応答速度を評価します。
* 処理内容: `fm_play_mml` 関数で短いフレーズを初期化して再生登録します。

## ベンチマーク結果の保存・出力

テストの経過時間（ティック、1ティック = 10ms）は `log_result()` 関数を介し、画面コンソールとログファイルの両方へ出力されます。
出力ファイル: `0:/bench.log`

> **Note:** PC-98のFAT12ファイルシステムにおける文字制約（8文字.3文字以内）に準拠した名前が使用されています。

## ビルドおよび実行

ベンチマークプログラムはOS32に組み込み済みの `Makefile` から自動でビルドおよび `images/os32_boot.d88` イメージに統合されます。

```bash
# 実行手順 (シェル内)
> exec bench.bin
```
