# Phase 2: シングルタスク切り替えによるアドレス空間の保護

## 概要
プログラムを実行・ロードする際、親プログラム（シェル等）の物理メモリやページマッピングを上書きして破壊しないよう、ページディレクトリの切り替えによる物理メモリ保持機構（コンテキストスイッチ）を導入する。

## タスクリスト
- [ ] 新しいプログラムをロードする際、カーネルから新規の物理メモリ領域（ページ群）を動的に確保する。
- [ ] 対象のプログラム実行用ページディレクトリ（マッピング）を用意し、実行仮想アドレス `EXEC_LOAD_ADDR`（`0x400000`）へマッピングを差し替える。
- [ ] 呼び出し元（親プロセス）の物理メモリはそのまま維持し、マッピングのみを外すことで安全に保留状態にする（スワップは行わない）。
- [ ] 子プロセスが終了した時点で、退避していた親プロセスのページディレクトリ設定へと切り替え戻し、瞬時に元の状態に復帰できるようにする。
