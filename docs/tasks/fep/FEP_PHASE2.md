# FEP Phase 2: OS32 辞書バイナリ読み込み・検索エンジンの開発

## 目的
OS32の限られたメモリ・リソース上で、ディスク(HDD)上の`.dic`ファイルから効率よく候補単語を検索し取り出すFEPエンジンのコアロジックを開発する。最初はユーザープログラムレベル（テストアプリ）として構築・検証する。

## 前提条件
- Phase 1 で生成された `.dic` バイナリが利用可能であること
- `.dic` は「ファイルヘッダ（16B）＋ブロックインデックス＋データブロック部」の3層構造
- 実機のユーザー利用可能メモリは約2MB（インデックス部のオンメモリ常駐が必須）
- ファイルI/OはKernelAPIの `sys_open / sys_read / sys_lseek / sys_close` を利用

## 設計上の注意点
- **既存SKKとの関係**: SKK (`programs/skk/skk_dict.c`) はLZSS圧縮されたテキスト辞書を全文メモリに展開して二分探索する方式。FEPでは辞書サイズが桁違いに大きいため、ブロック単位のディスクI/Oが必須。ただし`skk_dict_search()`の二分探索ロジックは参考にできる。
- **メモリ制約**: データブロックの作業バッファは1つ分だけ確保し、使い回す設計（同時に複数ブロックをロードしない）。
- **エンディアン**: i386はリトルエンディアンなので、`WordMeta32` はホスト(コンパイラ)・ターゲット(i386) 間でエンディアン変換は不要。
