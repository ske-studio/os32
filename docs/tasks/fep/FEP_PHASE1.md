# FEP Phase 1: ホスト側辞書コンパイラの開発と検討

## 目的
IPADIC等の辞書リソースから、OS32環境上で効率的に検索・ロード可能な独自バイナリ形式(`.dic`)の辞書ファイルを構築するための開発ツールを実装する。

## 前提条件
- IPADICのCSVは `assets/ipadic/` にダウンロード済み（EUC-JP）
- ファイルのカラム構成: `表層形, 左文脈ID, 右文脈ID, コスト, 品詞, 品詞細分類1, ..., 原形, 読み(カタカナ), 発音`
- `Verb.csv` / `Adj.csv` には同一単語の全活用形が別行として含まれる（約16万行）

## 設計上の留意点
- **読みの文字コード**: IPADICの読みは全角カタカナだが、FEPでのユーザー入力はひらがなで行う。辞書をひらがなで統一保存するか、検索時にカタカナ化してから照合するかの方針を実装前に決定する必要がある。
- **WordMeta32のビットレイアウト**: `yomi_len(5bit)`は最大31バイトだが、UTF-8のひらがな/カタカナは1文字3バイトのため、最大10文字程度が上限。長い読みの語彙は切り捨てるか、レイアウトの再検討が必要。
- **コスト値のスケーリング**: IPADICのコスト値範囲（概ね-3000〜30000程度）を12bit（0〜4095）に圧縮するマッピング関数の設計。
