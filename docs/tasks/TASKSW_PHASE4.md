# Phase 4: シェルロジックの適正化と動作検証

## 概要
ネスト呼び出しとタスク切り替えの基盤（Phase 1〜3）が完成したのを受け、シェルが外部コマンド終了後に「リロード（再起動）」に頼ることなく、そのままクリーンに復帰できるように処理を適正化する。

## タスクリスト
- [ ] `kernel.c` 側の `kernel_main` にあるシェルの無限ループを、システム起動（およびクラッシュ時のフォールバック）のみのシンプルな処理に整理する。
- [ ] `shell.c` の `execute_command()` および関連ロジックを見直し、`sys_exec_run` 終了後に画面を保持したまま次のプロンプト表示・入力待ちへ進めるようにする。
- [ ] デモプログラム（`demo1.bin`など）を実行し、終了後に即座にシェルに戻ることを検証する。
- [ ] シェル上で複数のプログラムを利用し、履歴や状態が保持されること、タスク切り替えを伴うメモリリークやページフォルトが発生しないことを検証・デバッグする。
