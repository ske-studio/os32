# Phase 3: 共有メモリ空間 (Shared Memory) の導入と保護機構

## 概要
プロセス切り替え下においても、全プログラムから常に同じ物理メモリを参照できる静的な共有メモリ帯域（256KB）を定義し、16KBごとのブロック分割とページ保護による堅牢なIPC（プロセス間通信）を提供する。

## タスクリスト
- [ ] `memmap.h` を更新し、`0x200000` 付近などのカーネル予約領域に 256KB の共有メモリ空間を静的に確保する。
- [ ] 256KBを16KB単位で16個に区切る「固定長ブロック分割管理」を実装し、`sys_shm_alloc(N)` 等の新規 Kernel API を追加する。
- [ ] コピー完了時に対象ブロックの領域を Read-Only 属性へ保護・ロックする `sys_shm_lock(ptr)` APIを実装する。
- [ ] 永続的に何度でも読み出せる「連続ペースト」に対応しつつ、不要になった際に Read-Write へ戻す明示的な解放API `sys_shm_free(ptr)` を実装する。
- [ ] 共有メモリ空間（256KB）の直前および直後に ガードページ (NOT PRESENT) を設定し、オーバーフローによる他領域破壊を完全に防止する機能を追加する。
