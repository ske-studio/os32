# Phase 4: 拡張キー設定 (0x95h) — Windowsキー有効化

## 概要
UNDOCUMENTED io_kb.md の下りコマンド `0x95h` を使用し、PC-9801-119キーボードのWindowsキー等の拡張キーを有効化する。OS32のホットキーやランチャー機能への割り当てを可能にする。

## 背景
- PC-9801-119 のみWindowsキー（左Win/右Win/Appキー）を搭載
- デフォルトではこれらのキーは無効状態であり、0x95hコマンドで明示的に有効化する必要がある
- 有効化すると、押下時に通常のキースキャンコードとは異なる拡張コードが送信される

## 出典
- UNDOCUMENTED 9801/9821 Vol.2 io_kb.md — I/O 0041h WRITE コマンド `95h`
- 対象: PC-9801-98, PC-9801-119 のみ

## 95hコマンド仕様
```
送信: 95h xxh
  xx の詳細はUNDOCUMENTED参照
  PC-9801-119: Windowsキーの有効/無効を設定
  PC-9801-98:  拡張キーの設定
```

## タスクリスト
- [ ] Phase 3 の `kbd_identify()` で PC-9801-119 と判定された場合のみ実行
- [ ] `kbd_enable_winkeys()` 関数を実装
  - `kbd_send_cmd(0x95)` → 有効化パラメータ送信
- [ ] Windowsキーのスキャンコードを特定し、キーマップに追加
- [ ] OS32ホットキーとしての活用設計
  - 例: Win+Q → FEP切替、Win+E → エディタ起動 等

## 依存関係
- Phase 1, Phase 3 が先に必要
- 実機 + PC-9801-119キーボードが必要（NP21/Wでは確認不可能な可能性が高い）

## 注意事項
- PC-9801-119 限定のため、大多数の環境では不要
- NP21/Wが119キーボードをエミュレートしているかは不明
- 0x95hの詳細パラメータはUNDOCUMENTED資料でも完全には解析されていない部分がある
- 本フェーズは他の全フェーズが安定してから着手すべき低優先度タスク
