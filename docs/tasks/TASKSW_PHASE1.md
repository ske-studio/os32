# Phase 1: 実行状態のスタック管理化

## 概要
外部プログラムの呼び出し時（`sys_exec_run`）に、コンテキスト状態を保持する変数をネスト対応（スタック化）させ、子プロセス終了時に親プロセスへと正しく `longjmp` で戻れるようにする。

## タスクリスト
- [ ] `exec.c` の単一フラグ `is_exec_running` を、深度を表すカウンタ `exec_nest_level` に変更する。
- [ ] 単一のバッファだった `exec_kernel_jmpbuf` を配列化し（例: `MAX_EXEC_NEST = 4`）、深度ごとに保存できるようにする。
- [ ] 現在ローカル変数のため復帰時に不定になってしまう `guard_a`, `guard_b`, `sbrk_heap_limit` 等のメモリ・ヒープ管理変数も配列の構造体に格納し、復元時の状態不整合（ページフォルトの原因）を防ぐ。
- [ ] プログラム終了処理 `exec_exit` (sys_exit) 時に `exec_nest_level` を減算し、正しく自分を呼び出した階層（親プロセス）へ `exec_longjmp` する機能に修正する。
