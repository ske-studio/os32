# Phase 1: キーボードリピート速度制御 (0x9Ch)

## 概要
UNDOCUMENTED io_kb.md に記載の下りコマンド `0x9Ch` を使用し、キーボードのリピート速度（初回遅延と繰り返し速度）をソフトウェアから変更可能にする。VZエディタやシェルでのカーソル移動・文字削除の操作感を改善する。

## 背景
- 現在のOS32はBIOSデフォルトのリピート速度に依存しており、変更手段がない
- PC-9801-98以降の「新キーボード」は I/O 0041h WRITE で下りコマンドを受け付ける
- エディタでの長距離カーソル移動やDELキー長押しで体感差が大きい

## 出典
- UNDOCUMENTED 9801/9821 Vol.2 io_kb.md — I/O 0041h WRITE コマンド `9Ch`
- 対象: 新キーボード搭載機 (PC-9801-98, 106, 109, 114, 115, 116, 119等)

## 9Chコマンド仕様
```
送信: 9Ch DDh RRh
  DD = リピート開始遅延 (0:250ms, 1:500ms, 2:750ms, 3:1000ms)
  RR = リピート速度 (0:最速 ～ 31:最遅)
```

## タスクリスト
- [ ] `drivers/kbd.c` にキーボードへのコマンド送信関数 `kbd_send_cmd(u8 cmd)` を実装
  - 8251A TxRDY (I/O 0043h bit 0) を確認してから I/O 0041h に書き込む
  - タイムアウト処理を含める
- [ ] `kbd_set_repeat(u8 delay, u8 rate)` 関数を実装
  - `kbd_send_cmd(0x9C)` → `kbd_send_cmd(delay)` → `kbd_send_cmd(rate)` の3バイト送信
- [ ] `kernel_main()` の初期化シーケンスでデフォルト値 (delay=0, rate=8 程度) を設定
- [ ] KAPI に `kbd_set_repeat` を公開し、シェルの `keyrate` コマンドから変更可能にする
- [ ] NP21/Wでの動作確認 (新キーボードエミュレーション対応の確認)

## 注意事項
- 旧キーボード (初代～PC-9801M3) は下りコマンド非対応。送信しても無視される
- NP21/Wが下りコマンドをエミュレートしているかは要確認。未対応の場合は実機でのみ効果あり
- 8251Aのステータス確認なしに書き込むとデータ破壊の可能性あり
